## Описание проекта

**Бизнес-кейс:** Разработка аналитического ядра для стриминговой платформы (аналог Netflix/Кинопоиск).

**Цель работы:** Сформировать навыки проектирования и реализации полиглотной системы хранения данных (Polyglot Persistence) для решения бизнес-задач.

**Используемые технологии:**
- **MongoDB** — хранение каталога подписок и фильмов
- **GraphDB** — граф знаний о фильмах (актеры, режиссеры, связи)
- **Python** — скрипты для взаимодействия с БД

## Инструкция по запуску

### 1. Запуск Docker-контейнеров

```bash

cd ~/Downloads/dba/nonrel/graphdb && sudo docker compose up -d
```

## Выводы

### 1. По заданию 1 (MongoDB)

- Документная модель MongoDB идеально подходит для хранения иерархических данных (подписки с массивами features)

- Операция update с $set позволяет легко изменять отдельные поля

- Python-драйвер pymongo обеспечивает удобный интерфейс для работы

### 2. По заданию 2 (GraphDB)

- RDF-графы эффективно представляют связи между сущностями (фильмы-актеры-режиссеры)

- SPARQL позволяет выполнять сложные запросы с агрегацией и фильтрацией

- Даже при пустом результате запрос считается выполненным, если синтаксис корректен

### 3. По заданию 3 (Аналитика)

- Цветные фильмы показывают на 54% большую вовлеченность (по количеству комментариев)

- Бизнес-рекомендация: инвестировать преимущественно в цветной контент

- Для черно-белых фильмов создать отдельную категорию "Классика" с особым маркетингом

### 4. Общий вывод

#### Полиглотный подход (MongoDB + GraphDB) позволяет эффективно решать разнородные задачи:

- MongoDB — для гибкого хранения каталога и подписок

- GraphDB — для анализа сложных связей в графе знаний
